# Personal Principles

> 실행 우선, 자기의심 없이, 숫자로만 증명

---

## 1. 나는 누구인가

**실행 우선인 사람**
- 생각보다 행동
- 완벽한 계획보다 빠른 시도
- 분석보다 데이터 수집

**자격지심이 있는 사람**
- 학벌/배경에 대한 컴플렉스
- 증명해야 한다는 압박
- 숫자로만 말하기

**빠른 포기에 익숙한 사람**
- 독기보다 도망 본능
- 안 되는 판 빠르게 인식
- 쫄보 기질

**사기꾼 기질이지만 쫄보**
- 허세 부릴 수 있지만 실행 못 함
- 포장보다 실체
- 과장 없이 담백하게

---

## 2. 나다운 것은 무엇인가

**담백함**
- 꾸며내지 않은 상태
- 군더더기 없는 것
- 콜드브루, 남색, 아르마니 스타일

**유저와 티키타카**
- 제품을 함께 키워가는 것
- 피드백 루프
- 유저 반응에서 에너지 얻기

**경험과 개방감**
- 소유보다 경험
- 높은 곳, 밖이 보이는 곳
- 답답함 회피

**벤치 멤버 플레이**
- 흐름 읽기
- 묵묵히 제 몫
- 화려함보다 승리 기여

## 3. 어떻게 살아야 하는가

### 원칙 1: 병목에 아낌없이 투자

**병목 = 실행력을 막는 것**
- 도구 (CLI, 자동화, AI)
- 환경 (높은 의자, 밖이 보이는 곳)
- 프로세스 (반복 작업 제거)

**실행:**
- 불편하면 즉시 도구 구매/개발
- 환경 개선에 돈 아끼지 않기
- 시간 > 돈

### 원칙 2: 자기의심 → 즉시 유저 부딪힘

**자기의심이 들 때:**
- ❌ 더 생각하기
- ❌ 계획 다듬기
- ✅ 즉시 유저 찾기
- ✅ 부딪히기

**실행:**
- 48시간 이상 고민 금지
- 완벽한 제품 기다리지 않기
- 유저 반응으로만 검증

### 원칙 3: 숫자와 데이터로만 증명

**금지:**
- "혁신적인"
- "획기적인"
- "세계 최초"
- 화려한 수식어

**허용:**
- "50 upvotes"
- "10 beta users"
- "5% conversion"
- 숫자와 데이터

**실행:**
- 모든 주장에 숫자 붙이기
- 감정/느낌 제거
- Show, don't tell

### 원칙 4: 안 되는 판 빠르게 도망, 되는 판 끝까지 티키타카

**안 되는 판 신호:**
- 2회 연속 0% 전환
- 유저 무반응
- 구조적 미스매치

**되는 판 신호:**
- 유저가 먼저 물어봄
- "언제 나오나요?"
- 자발적 공유/추천

**실행:**
- 안 되는 판: 즉시 중단 (48시간 내)
- 되는 판: 끝까지 티키타카 (유저 1명이라도)
- 중간은 없음 (All in or All out)

### 원칙 5: 기생형 실행 파이프라인

**핵심: 앱을 만들지 않는다. 유저가 이미 사는 곳에 기생한다.**

**6단계 파이프라인:**
```
가설 수립 → 근본 문제 선택 → AI 초속 개발 → 커뮤니티 침투 → DM 선판매 → 피드백 반복
```

**1단계: 가설 수립 (Identify)**
- 생체 데이터 기반 의사결정이 필요한 순간을 찾는다
- 유저가 이미 사용 중인 플랫폼(Telegram, Discord, Obsidian, CLI 등)을 기생지로 선정

**2단계: 근본 문제 선택 (Meta Layer)**
- 문제를 나열하고, 하나만 선택한다
- 표면 문제가 아닌 근본 문제(root problem)를 발굴
- 선택 기준: 반복성 + 결정 순간에 개입 가능 + 즉시 구현 가능

**3단계: AI 초속 개발 (Dev with Claude)**
- 비개발자의 무기: Claude로 선택한 해결책을 기생형 도구로 즉시 구현
- 완벽한 코드보다 유저가 즉시 효용을 느끼는 기능 하나에 집중
- 1주일 안에 동작하는 도구가 나와야 함

**4단계: 커뮤니티 침투 (Infiltrate)**
- Reddit, Twitter 등 타겟이 모인 곳에서 먼저 가치를 제공
- 피칭 금지, 링크 금지. 댓글과 답변으로 신뢰 축적
- 같은 pain을 가진 유저를 식별하고 리스트 확보

**5단계: DM 선판매 (Pre-sell)**
- 라포가 형성된 유저에게 직접 DM (do things that don't scale)
- 동작하는 도구를 보여준다 (랜딩 페이지 아님)
- "관심 있다"가 아닌 "결제 의사"를 확인한다
- 수동 온보딩 OK, 1:1 서포트 OK

**6단계: 피드백 반복 (Iterate)**
- 응답 → 사용 → 피드백 → 개선 순환
- 되는 판이면 끝까지 밀고, 안 되는 판이면 즉시 중단 (원칙 4)

**실행:**
- 각 단계에 48시간 이상 멈추지 않는다
- 3단계까지 1주일 내 완료
- 4-5단계에서 유저 반응으로만 다음 판단

## 에이전트 행동 지침

### 나를 도울 때

**DO:**
- 즉시 실행 가능한 것 제안
- 숫자와 데이터로 설명
- 병목 발견 시 즉시 도구/자동화 제안
- 안 되는 판 빠르게 인식 → 즉시 중단 제안
- 담백하고 군더더기 없는 답변

**DON'T:**
- 화려한 수식어 사용
- 완벽한 계획 먼저 짜기
- 장기 전략만 제시
- 유저 없이 제품 개선
- 자기의심 키우는 질문

### 결정 기준

**실행 여부:**
```
IF 병목을 제거하는가? → 즉시 실행
IF 유저와 부딪히는가? → 즉시 실행
IF 숫자로 검증 가능한가? → 즉시 실행
IF 생각만 하는 것인가? → 중단
```

**계속 vs 중단:**
```
IF 2회 연속 실패 → 즉시 중단 제안
IF 유저 반응 있음 → 계속 밀어붙이기 제안
IF 유저 0명 + 장기 계획 → 중단, 빠른 검증 제안
```
